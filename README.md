## 🧠 How It Works

```
User Profile (age/weight/height/gender/goal/activity)
         ↓
  BMR + TDEE Calculator (Mifflin-St Jeor)
         ↓
  Target Macros (calories/protein/carbs/fat)
         ↓
  Claude AI (claude-sonnet-4-20250514)
  + Live USDA FoodData Central
         ↓
  5-Meal Diet Plan  +  Weekly Workout Plan
  saved to PostgreSQL
         ↓
  React Dashboard (macros, charts, progress tracker)
```

## 🚀 Quick Start (Local Dev)

### Prerequisites
- Docker + Docker Compose
- Node.js 20+
- Python 3.12+
- PostgreSQL 16 (or use Docker)

### 1. Clone & configure

```bash
git clone https://github.com/yourusername/fitai.git
cd fitai

# Backend env
cp backend/.env.example backend/.env
# Edit backend/.env and add your keys:
# ANTHROPIC_API_KEY=sk-ant-...
# USDA_API_KEY=...  (get free key at https://api.nal.usda.gov)
```

### 2. Run with Docker Compose (recommended)

```bash
docker-compose up --build
```

| Service   | URL                          |
|-----------|------------------------------|
| Frontend  | http://localhost:5173        |
| Backend   | http://localhost:8000        |
| API Docs  | http://localhost:8000/docs   |
| DB        | localhost:5432               |

### 3. Run manually (without Docker)

**Backend:**
```bash
cd backend
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env        # fill in your keys

# Start PostgreSQL first, then:
alembic upgrade head
uvicorn app.main:app --reload --port 8000
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

## 🔁 CI/CD Pipeline

The GitHub Actions pipeline runs on every push:

```
Push to main/develop
    │
    ├── backend-ci
    │     ├── Lint (ruff)
    │     ├── Unit tests (pytest)
    │     └── Coverage report (Codecov)
    │
    ├── frontend-ci
    │     ├── Lint (ESLint)
    │     ├── Type check (tsc)
    │     └── Build (Vite)
    │
    ├── docker-build  [main only]
    │     ├── Build & push backend image → ghcr.io
    │     └── Build & push frontend image → ghcr.io
    │
    ├── deploy  [main only]
    │     ├── Backend → Railway
    │     └── Frontend → Vercel
    │
    └── smoke-test
          └── Health check API
```

### Required GitHub Secrets

| Secret | Description |
|--------|-------------|
| `ANTHROPIC_API_KEY` | Claude API key |
| `RAILWAY_TOKEN` | Railway deploy token |
| `VERCEL_TOKEN` | Vercel deploy token |
| `VERCEL_ORG_ID` | Vercel org ID |
| `VERCEL_PROJECT_ID` | Vercel project ID |

## 🌐 Free Deployment (Recommended)

### Backend → Railway (free tier)
1. Go to https://railway.app → New Project → Deploy from GitHub
2. Select the `backend` folder
3. Add env vars from `.env`
4. Add a PostgreSQL plugin

### Frontend → Vercel (free tier)
1. Go to https://vercel.com → New Project → Import from GitHub
2. Select the `frontend` folder
3. Set `VITE_API_URL=https://your-railway-backend.railway.app`
